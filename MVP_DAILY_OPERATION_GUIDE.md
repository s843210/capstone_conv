# MVP Daily Operation Guide

## 목적

이 문서는 편의점 재고/발주 추천 MVP를 하루 단위로 운영할 때 관리자가 꼭 해야 하는 작업 순서를 정리한다.

## 매일 필수 작업 순서

### 1. 어제 판매 데이터 업로드

설정 화면의 `판매 업로드`에서 어제 판매 파일을 업로드한다.

예를 들어 오늘이 `2026-05-12`라면 `2026-05-11` 판매 데이터를 업로드한다.

이 데이터는 `daily_sales`에 저장되며, AI가 `lag_1`, `lag_7`, `rolling_mean_7`, `rolling_mean_14`, `rolling_mean_28` 같은 판매 추세 feature를 만들 때 사용한다.

### 2. 오늘 현재고 업로드

설정 화면의 `재고 업로드`에서 오늘 기준 현재고 파일을 업로드한다.

업로드 결과는 다음 DB 값에 반영된다.

```text
product.current_stock
inventory_snapshot
```

AI 추천은 실행 시점의 `product.current_stock`을 기준으로 현재 재고를 판단한다.

### 3. 오늘 날씨/Context 갱신

설정 화면의 `날씨/Context 갱신`에서 대상 날짜를 오늘로 설정하고 실행한다.

예를 들어 오늘이 `2026-05-12`라면 대상 날짜는 `2026-05-12`로 입력한다.

이 작업은 `daily_context`에 다음 값을 반영한다.

```text
avg_temp_c
precipitation_mm
is_rain
is_holiday
academic_event
building_headcount
```

주의: 현재 날씨 API는 오늘 또는 미래 예보 조회용이다. 과거 날짜는 실패할 수 있다.

### 4. AI 예측 실행

설정 화면의 `AI 예측 실행`에서 대상 날짜를 오늘로 설정하고 실행한다.

예를 들어 오늘이 `2026-05-12`라면 대상 날짜는 `2026-05-12`로 입력한다.

실행 흐름은 다음과 같다.

```text
Spring
-> FastAPI
-> DB 조회
-> monthly_v2 feature 생성
-> random_forest_monthly_v2_model.pkl 예측
-> ai_prediction 저장
```

AI는 기본적으로 다음 데이터를 사용한다.

```text
daily_sales: 대상 날짜 전날까지의 판매 이력
daily_context: 대상 날짜의 날씨/학사/공휴일/유동인구
product: 현재고와 상품 정보
```

### 5. 대시보드 추천 확인

대시보드에서 최신 `ai_prediction.target_date` 기준 추천 발주 목록을 확인한다.

추천 수량은 자동 계산 결과이므로, 최종 발주 전 관리자가 실제 상황을 보고 조정할 수 있다.

## 매일 하지 않아도 되는 작업

### 상품 마스터 업로드

상품분류기준표가 바뀌었거나, 카테고리 매칭 기준을 새로 반영해야 할 때만 실행한다.

### monthly_v2 processed bulk import

초기 DB 세팅 또는 서버를 새로 띄울 때 과거 학습용 원천 데이터를 한 번에 적재할 때만 실행한다.

### 학사일정 업로드

학사일정 CSV나 유동인구 기준 파일이 바뀌었을 때만 실행한다.

### 재학습

매일 실행할 필요 없다. 판매 데이터가 충분히 쌓인 뒤 주기적으로 실행한다.

## 하루 운영 체크리스트

```text
[ ] 어제 판매 데이터 업로드
[ ] 오늘 현재고 업로드
[ ] 오늘 날씨/Context 갱신
[ ] 오늘 AI 예측 실행
[ ] 대시보드 추천 확인
```

## 날짜 기준 예시

오늘이 `2026-05-12`인 경우:

```text
업로드할 판매 데이터: 2026-05-11
업로드할 현재고: 2026-05-12 기준 현재고
날씨/Context 대상 날짜: 2026-05-12
AI 예측 대상 날짜: 2026-05-12
```

AI는 `2026-05-11`까지의 판매 이력과 `2026-05-12`의 context, 실행 시점의 현재고를 사용해 `2026-05-12` 추천을 만든다.
